# Missão: Site B2B Doris

## Objetivo
Desenvolver o site B2B da Doris no Webflow com base nos wireframes e protótipos do Figma.

## Stack
- **CMS/Builder:** Webflow (já integrado via MCP)
- **Design:** Figma (MCP conectado)
- **Brandbook:** `docs/Brandbook 2026/` — consultar sempre antes de decisões visuais

## Figma

- **Protótipos:** https://www.figma.com/design/o7KuIO82uHI5Z2HT5lh4ns/-B2B--Wireframe---Novo-Site-Doris?node-id=49-98&p=f&t=oyrLYeb9fDV1Jo38-0

## Contexto do produto
- Doris é uma fashion tech brasileira (B2C app + parcerias B2B com varejistas de moda)
- O site B2B é voltado para varejistas, marcas e parceiros — não para o consumidor final
- Tom: profissional, sofisticado, orientado a negócios — seguir brandbook
- Tagline da marca: "O futuro da moda é você."

## Restrições críticas
- ⚠️ O projeto Webflow já possui **header e footer desenvolvidos** — NUNCA modificar, substituir ou recriar esses elementos
- Sempre preservar o header e footer existentes ao adicionar ou editar qualquer página
- Em caso de dúvida sobre qualquer elemento do projeto Webflow, **perguntar antes de agir**
- Tentativas anteriores de construir o site sem essa restrição quebraram o header e footer — isso não pode se repetir

## Modo de trabalho
- Perguntar sempre que houver dúvida antes de executar
- Confirmar com o usuário antes de qualquer alteração estrutural no projeto Webflow
- Preferir ações incrementais e reversíveis

## Instruções para os agentes
1. Ler os wireframes e protótipos no Figma antes de qualquer implementação
2. Consultar o brandbook em `docs/Brandbook 2026/` para cores, tipografia e identidade visual
3. Implementar no Webflow via MCP — respeitando obrigatoriamente o header e footer existentes
4. Seguir as regras de voz da Doris: clara, sofisticada, pessoal — nunca publicitária
5. Preferir ações incrementais — uma seção por vez, com validação antes de avançar
